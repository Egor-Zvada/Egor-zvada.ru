# Service Error Pages

Готовый набор статических страниц ошибок в стиле terminal / mono / egor_zvada.

Сделано под сайт, приложение или сервис. В этом репозитории пакет лежит в `/error-pages/`
и может переиспользоваться в других проектах без привязки к основной админке.

## Страницы

- `400.html` — неверный запрос
- `401.html` — нужна авторизация
- `403.html` — доступ запрещён
- `404.html` — страница не найдена
- `405.html` — метод не разрешён
- `408.html` — истекло время запроса
- `409.html` — конфликт
- `410.html` — ресурс удалён
- `418.html` — шуточная пасхалка "я чайник"
- `422.html` — данные не удалось обработать
- `429.html` — слишком много запросов
- `451.html` — доступ ограничен по юридическим причинам
- `500.html` — внутренняя ошибка сервера
- `501.html` — не реализовано
- `502.html` — плохой шлюз
- `503.html` — сервис недоступен
- `504.html` — тайм-аут шлюза

`index.html` равен `404.html`.
`666.html` оставлен как отдельная пасхалка и не подключается к nginx по умолчанию.
Для прямой ссылки на "я чайник" есть `/teapot/`.

## Структура

```text
error-pages/
├── index.html
├── 400.html
├── ...
├── nginx.example.conf
└── assets/
    ├── css/style.css
    └── js/app.js
```

## nginx пример

```nginx
include /var/www/egor-zvada.ru/error-pages/nginx.example.conf;
```

`/teapot` и `/teapot/` в примере возвращают HTTP 418 и показывают `418.html`.
Если `666.html` понадобится подключить как пасхалку, лучше делать это отдельным `location`.

## Локальный просмотр

```bash
python3 -m http.server 8080
```

Потом:

```text
http://127.0.0.1:8080/error-pages/404.html
http://127.0.0.1:8080/error-pages/418.html
http://127.0.0.1:8080/error-pages/666.html
http://127.0.0.1:8080/teapot/
```
